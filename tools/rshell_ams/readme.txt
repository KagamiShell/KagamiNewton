
kshell_ams - утилита для интеграции Runpad/RunpadPro и Astalavista AMS.
Интеграция представляет собой выдачу информации про сеанс пользователя на рабочий стол шелла.

Замечания:
* Утилита работает совместно как с RunpadPro, так и с Runpad.
* Для добавления программы в автозагрузку текущего пользователя (HKEY_CURRENT_USER) используйте: 
  kshell_ams.exe /autorunCU
* Для добавления программы в автозагрузку всех пользователей (HKEY_LOCAL_MACHINE) используйте: 
  kshell_ams.exe /autorunLM
  (будут нужны права администратора системы!)
* Для удаления программы из автозагрузки используйте:
  kshell_ams.exe /uninstall
  (для удаления из HKEY_LOCAL_MACHINE будут нужны права администратора системы!)
* Описание настроек и примеры настроек находятся в файле kshell_ams.ini

Рекомендации:
* Можно также добавить kshell_ams.exe в опцию "Восстановление клиента" в настройках Runpad.
* Для большей безопасности добавляйте утилиту в автозагрузку HKEY_LOCAL_MACHINE,
  а работайте под ограниченным пользователем.
* Более удобный и самый безопасный способ - добавить программу в автозагрузку 
  на вкладке настроек шелла - "Обслуживание: Автозагрузка".
* Для большей безопасности файл настроек kshell_ams.ini храните в папке Windows 
  (там же можно хранить и kshell_ams.exe).
* Для быстрого обновления/добавления файла kshell_ams.ini (kshell_ams.exe) на все машины 
  при его хранении в папке Windows используйте средства автоматического обновления RunpadPro:
  1) в программе администратора в папке "C:\Program Files\Runpad Pro Admin\update\client\"
     создайте папку "WINDIR" и в нее перепишите ваш kshell_ams.ini (kshell_ams.exe);
  2) запустите программу начальной конфигурации БД RunpadPro - rsdbconf.exe, чтобы занести
     этот файл (файлы) в SQL-базу;
  3) в программе оператора на выбранных машинах выполните обновление клиентского шелла,
     после чего ваш файл kshell_ams.ini (kshell_ams.exe) попадет в папку \Windows 
     каждого клиентского компьютера.
  Также можно использовать и удаленный файловый менеджер шелла для этих целей.
